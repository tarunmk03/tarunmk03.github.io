Splunk Home SOC Lab

Built a Home SOC Lab using Splunk Enterprise to simulate real-world Security Operations Center (SOC) monitoring and detection workflows. The lab focuses on Windows Security Event monitoring, authentication analysis, SPL query development and threat detection.

How I Built It

  • Installed Splunk Enterprise on a Windows system.
  • Collected Windows Security, System and Application logs.
  • Verified log ingestion and field extraction.
  • Generated successful and failed authentication events.
  • Created SPL queries for authentication monitoring.
  • Developed brute-force detection logic using failed login events.
  • Performed log analysis and security investigations.

Technologies Used

Splunk Enterprise • Windows Event Logs • SPL (Search Processing Language) • Authentication Monitoring • Threat Detection • Security Monitoring

01. Log Ingestion Overview

Splunk Enterprise successfully ingesting Windows Security, System and Application event logs for centralized monitoring.

02. Windows Security Log Collection

Verification of Windows Security Event collection and field extraction within Splunk.

03. Successful Login Monitoring (Event ID 4624)

Authentication monitoring using Windows Event ID 4624 to analyze successful login activity.

04. Failed Login Monitoring (Event ID 4625)

Detection and investigation of failed authentication attempts using Windows Security Event ID 4625.

05. Brute Force Detection Use Case

Developed an SPL-based brute-force detection use case identifying multiple failed login attempts within a five-minute period.

Key SPL Queries

Successful Login Monitoring

index=* sourcetype=WinEventLog:Security EventCode=4624 | stats count by Account_Name

Failed Login Monitoring

index=* sourcetype=WinEventLog:Security EventCode=4625 | stats count by Account_Name

Brute Force Detection

index=* sourcetype=WinEventLog:Security EventCode=4625 | bucket _time span=5m | stats count by _time Account_Name | where count >= 5
